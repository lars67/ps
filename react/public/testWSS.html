Technical Documentation

This test page is designed to help test WebSocket connections to the server in both HTTP and HTTPS modes.

Server Configuration

The server is configured to run in dual-mode:

  • Production Mode: Uses HTTPS with SSL certificates on standard ports (3331-3334)
  • Development Mode: Uses HTTP without certificates on high ports (13331-13334)

Implementation Details

The server automatically detects if SSL certificates are available:

  • If certificates are available, it uses HTTPS as the primary protocol on standard ports
  • If certificates are not available, it falls back to HTTP on standard ports
  • It always starts the secondary protocol on high ports (10000 higher than standard)

Connection URLs

Use these URL formats in your application:

  • Production: wss://yourdomain.com:3331/ps2l/
  • Development: ws://localhost:13331/ps2l/
