PS2Dox Reference Manual - Portfolios

Portfolios

Portfolios in PS2Dox represent collections of financial instruments or assets.

Portfolio Operations

1. Create Portfolio

Command:

{
  "command": "portfolios.add",
  "name": "Growth Portfolio",
  "description": "High-risk, high-reward strategy",
  "currency": "USD",
  "baseInstrument": "SPY"
}

Parameters:

Parameter Description Required
name Name of the portfolio Yes
description Description of the portfolio strategy No
currency Base currency for the portfolio Yes
baseInstrument Benchmark instrument for performance comparison Yes

Output: Returns the newly created portfolio object.

Help Text: Creates a new portfolio with the specified details. The baseInstrument is used as a benchmark for performance calculations.

2. List Portfolios

Command:

{
  "command": "portfolios.list",
  "filter": {"currency": "USD"}
}

Parameters:

Parameter Description Required
filter Object with portfolio fields to filter results No

Output: Returns an array of portfolio objects matching the filter criteria.

Help Text: Retrieves a list of portfolios. You can use the filter to narrow down the results, for example, to show only portfolios in a specific currency.

3. Update Portfolio

Command:

{
  "command": "portfolios.update",
  "_id": "portfolio_id",
  "description": "Updated portfolio description"
}

Parameters:

Parameter Description Required
_id ID of the portfolio to update Yes
name, description, currency, baseInstrument Fields to update At least one

Output: Returns the updated portfolio object.

Help Text: Modifies an existing portfolio. You can update various fields such as name, description, or baseInstrument.

4. Remove Portfolio

Command:

{
  "command": "portfolios.remove",
  "_id": "portfolio_id"
}

Parameters:

Parameter Description Required
_id ID of the portfolio to remove Yes

Output: Returns a success message or the removed portfolio object.

Help Text: Deletes a portfolio from the system. This action is irreversible and will also remove all associated trades and performance data.

5. Get Portfolio Performance

Command:

{
  "command": "portfolios.getPerformance",
  "_id": "portfolio_id",
  "startDate": "2023-01-01",
  "endDate": "2023-12-31"
}

Parameters:

Parameter Description Required
_id ID of the portfolio Yes
startDate Start date for performance calculation (YYYY-MM-DD) Yes
endDate End date for performance calculation (YYYY-MM-DD) Yes

Output: Returns an object with performance metrics including returns, volatility, and comparison to the baseInstrument.

Help Text: Calculates and returns the performance metrics for a specified portfolio over a given date range. This includes returns, volatility, and comparison to the baseInstrument.
